# 🚨 قواعد إلزامية — اقرأ هذا قبل أي مهمة

## 1. قبل أي تعديل: قدّم خطة وانتظر الموافقة
لا تعدّل كودًا قبل عرض ما ستغيره والحصول على "نعم".

## 2. لا رأي بلا دليل
لا تقل "جاهز" أو "لا مشاكل" بدون:
- الأمر الذي شغّلته
- ناتجه الحرفي (status code / grep result)

## 3. افترض أن الكود فيه خطأ
مهمتك إيجاد الأخطاء، لا إثبات عدم وجودها.

## 4. الإقرار بالجهل إلزامي
في كل تقرير، اذكر "ما لم أفحصه" مع السبب.

## 5. التحقق قبل كل commit
شغّل: npx tsc --noEmit && npx vitest run && npm run build
أي فشل = توقف.

## 6. المهام الحساسة تحتاج مراجعة خارجية
أمن/انتخابات/بيانات شخصية ← أوصِ بمراجعة من وكيل آخر.

## 7. لا تقول "جاهز للنشر" إلا بتحقق ALL:
- 0 Critical + 0 High غير موثّقة
- بوابات الجودة PASS بالدليل
- قسم "ما لم يُفحص" مملوء
