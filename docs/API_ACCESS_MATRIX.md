# مصفوفة التحكم في الوصول وصلاحيات واجهات البرمجة (API Access Control Matrix)

توثق هذه الصفحة مصفوفة الصلاحيات المفروضة في منظومة الماكينة الانتخابية، لضمان حماية خصوصية بيانات الناخبين والحد من تسريب الهويات الشخصية (PII).

---

## 2. سياسة تقنيع الهوية الشخصية (PII Masking) لـ OBSERVER

عندما يسترجع مستخدم بدور `OBSERVER` بيانات الناخبين أو المفاتيح من خلال مسارات القوائم أو البحث، يتم تطبيق القيود التالية بشكل آلي في طبقة العرض (DTO):

1. **الاسم الكامل**: يظهر الاسم الأول فقط مع إخفاء باقي الاسم بـ `***` (مثال: `احمد ***`).
2. **رقم الهاتف**: تُقنع الأرقام الوسطى من الهاتف (مثال: `077****111`).
3. **الرقم الوطني**: يُرجع القيمة المقنعة `***` بالكامل.
4. **المعلومات الخاصة (المهنة، التحصيل، الحالة الاجتماعية، حجم العائلة، خط العرض/الطول، الملاحظات)**: تُرجع كـ `null` لـ `OBSERVER` لحظر جمع البيانات التفصيلية.
5. **البيانات المالية للمفتاح (الموازنة، المصاريف، تكلفة الصوت)**: تُرجع بقيمة صفر `0` لـ `OBSERVER`.

---

## 3. محددات الأمان الإضافية

- **فشل مغلق افتراضياً (Fail-Closed)**: في حال عدم ربط مستخدم من نوع `KEY_USER` بمفتاح انتخابي صريح، يتم إعطاؤه نطاق `"none"` افتراضياً، مما يمنعه من رؤية أو تعديل أي ناخبين أو مهام.
- **منع نقل الناخبين**: يُقفل حقل `keyId` للناخبين عند قيام `KEY_USER` بالتحديث لمنع نقل ناخب من نطاق مفتاح إلى مفتاح آخر.
- **إبطال الجلسات الفوري**: عند تغيير دور مستخدم أو تغيير كلمة مروره، يتم تحديث `tokenIssuedBefore` لإبطال جميع الجلسات الحالية الصادرة قبل هذا الوقت فوراً.
