## القسم 1: المبادئ غير القابلة للتفاوض (محفوظة كاملة من المصدر)

1. **لا ادعاء بلا دليل** — كل PASS مرتبط بأمر فعلي ونتيجته الحرفية.
2. **أقل صلاحية** — كل دور يرى وينفّذ أقل قدر يلزم فقط؛ الحالة المشكوك فيها تُرفض افتراضياً.
3. **الفشل المغلق (Fail-Closed)** — تعذّر التحقق من الجلسة/DB يمنع العملية الحساسة، لا Grace Mode للكتابة.
4. **فصل البيئات** — لا بيانات حقيقية في التطوير، لا نشر من جهاز شخصي غير موثّق.
5. **قابلية الرجوع** — لا نشر بلا نسخة احتياطية قابلة للاستعادة فعلياً وخطة Rollback مجرَّبة.
6. **الخصوصية بالتصميم** — تقليل جمع الهوية/الهاتف، تقنيعها عند العرض، منع ظهورها في السجلات.
7. **المراجعة المستقلة** — أي تغيير أمني أو متعلق بالبيانات يراجعه طرف لم يكتب الكود.
8. **الإصدار المجمَّد** — كل التقارير تشير إلى Git SHA واحد؛ أي تعديل بعد الفحص يُعيد البوابات المتأثرة.

## القسم 2: بوابات الكود الإلزامية (قبل كل commit/دمج)

```bash
npx tsc --noEmit                       # صفر أخطاء
npx eslint . --max-warnings=0          # صفر تحذيرات
npx vitest run                         # 100% نجاح
npm run build                          # نجاح كامل
node scripts/verify-security.js        # كل الفحوصات PASS، بما فيها تاريخ Git
npm audit --omit=dev --audit-level=high # 0 Critical/High بلا استثناءات
git ls-files | grep -E "^\.env"        # فارغ
git grep -nEi "(password|secret|token).*(example|changeme|hardcoded)" -- scripts/ src/ prisma/ docs/  # راجع كل نتيجة
```

**تنبيه من الوثيقة الأصلية (محفوظ):** نجاح `verify-security.js` مساعد فقط، ولا يعوّض مراجعة
RBAC أو فحص تاريخ الأسرار في Git الكامل (وليس آخر commit فقط).

### 2.1 فحص من Clone نظيف (فجوة حقيقية — لم تُنفَّذ بعد)
كل فحوصاتنا حتى الآن كانت على مجلد عمل قديم. قبل أي إصدار فعلي:
```bash
git clone <repo> /tmp/clean-check && cd /tmp/clean-check
npm ci && npx tsc --noEmit && npx vitest run && npm run build
```
نجاح محلي على مجلد قديم **لا يكفي**.

**متبقٍّ:** تعميم نفس المصفوفة على بقية المسارات الحساسة (voters تمّت جزئياً، tasks، services، sms، إلخ).

### 3.2 قواعد البيانات الشخصية (محفوظة)
- يُحظر `nationalId`/`phone`/`email`/كلمات المرور/`DATABASE_URL`/`JWT` في السجلات أو رسائل الخطأ.
- **✅ جديد اليوم:** تعقيم CSV/Excel ضد Formula Injection (`=`,`+`,`-`,`@`) — طُبِّق في `export/csv` و`export` لكلا المشروعين.

## القسم 5: النسخ الاحتياطي والتعافي

| البند | الحالة الحالية | المطلوب قبل أول إطلاق فعلي |
|---|---|---|
| وجود Backup | موجود ومشفّر (`src/lib/backup.ts`) | ربط مخزن دائم خارج دورة حياة الحاوية |
| **تجربة استعادة فعلية** | ❌ لم تُختبر إطلاقاً | **إلزامية غير قابلة للتفاوض** قبل أي بيانات حقيقية جديدة |
| RPO/RTO | غير مُعتمد رسمياً | يُقترح RPO ≤ 15 دقيقة، RTO ≤ ساعتين، يُثبَت بتجربة فعلية |

### سيناريو تجربة الاستعادة (يُنفَّذ مرة واحدة على الأقل قبل الإطلاق)
1. اختيار نسخة احتياطية معروفة، تسجيل وقتها ومعرّفها.
2. استعادتها في قاعدة **معزولة تماماً عن الإنتاج**.
3. تشغيل `migrate status` وفحوص سلامة البيانات والعلاقات.
4. تسجيل الدخول واختبار المسارات الأساسية على القاعدة المستعادة.
5. توثيق الزمن الفعلي والبيانات المفقودة إن وجدت.

## القسم 6: مخاطر متبقية مقبولة صراحة (مؤجَّلة — ليست بوابات NO-GO حالياً)

هذا القسم يطبّق مبدأ §21 من الوثيقة الأصلية حرفياً: "أي استثناء يُسجَّل بالسبب والنطاق
والخطر والضوابط البديلة والمالك وتاريخ الانتهاء" — بدل تجاهل هذه البنود صامتاً.

| البند | السبب | الضابط البديل الحالي | تاريخ مستهدف |
|---|---|---|---|
| MFA لحساب ADMIN | يحتاج تطويراً كاملاً (TOTP/Recovery Codes) غير موجود | كلمة مرور قوية + bcrypt(12) + rate limiting | غير محدد — يقرره المالك |
| WAF/حماية Edge | يحتاج مزوداً خارجياً (Cloudflare) وقراراً مالياً | Rate limiting على مستوى التطبيق + rueSHA headers | غير محدد |
| HA متعدد Instances | Railway single-instance حالياً؛ تكلفة مضاعفة غير مبررة لحجم الحملة | Health check + auto-restart عبر `restartPolicyMaxRetries` في railway.toml | غير مطبَّق — الحجم الحالي لا يبرره |
| اختبار اختراق مستقل مدفوع | تكلفة مالية، يحتاج مزوداً خارجياً | مراجعة كود داخلية + الفحوصات الآلية الحالية | يُقرَّر عند اقتراب أي حدث حساس فعلي |
| DAST (OWASP ZAP)، SBOM، Trivy/Grype | لم تُنفذ في التدقيق المحلي | `npm audit` + مراجعة يدوية للتبعيات | قبل الاعتماد الميداني |
| اختبار حمل كامل (Load/Soak) | لا أداة مثبّتة | فحص يدوي لعدد محدود من الطلبات المتزامنة | يُرقّى عند نمو عدد المستخدمين الفعلي |
