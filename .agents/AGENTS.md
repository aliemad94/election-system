# قواعد وإرشادات العمل على المشروع (Project-Scoped Rules)

* **الحفاظ على سلامة واستقرار النظام (Code Integrity & Stability)**: 
  عند إضافة أي ميزة جديدة، أو تعديل، أو تحسين، أو إصلاح لأي عطل، يجب الحفاظ بشكل صارم على جميع الوظائف البرمجية السليمة القائمة في المشروع. يُمنع منعاً باتاً حذف أو إهمال أي كود، منطق عمل (Business Logic)، أو إعدادات سابقة تؤثر سلباً على استقرار النظام أو وظيفته.
  
* **التحقق المستمر (Continuous Verification)**:
  قبل إنهاء أي مهمة، يجب تشغيل اختبارات الوحدة (`npx vitest run`) والتحقق من سلامة البناء الكلي للمشروع (`npm run build`) لضمان عدم حدوث أي تراجع (Regressions).
